INTRODUCTION - WHAT IS SCREENGRAM?
================================================================================

ScreenGram یک سیستم جامع مدیریت و کنترل نمایشگرهای دیجیتال (Digital Signage) است که 
برای مدیریت، نظارت و کنترل از راه دور نمایشگرهای فیزیکی در محیط‌های مختلف طراحی شده است.

هدف اصلی سیستم:
--------------
ScreenGram به سازمان‌ها و کسب‌وکارها امکان می‌دهد تا:
• نمایشگرهای دیجیتال خود را از یک پنل مرکزی مدیریت کنند
• محتوا (تصاویر، ویدیوها، متن‌ها) را به صورت خودکار روی نمایشگرها منتشر کنند
• برنامه‌های زمانی برای نمایش محتوا تنظیم کنند
• وضعیت نمایشگرها را به صورت لحظه‌ای نظارت کنند
• دستورات را از راه دور به نمایشگرها ارسال کنند (ریستارت، تغییر قالب، همگام‌سازی)

کاربردهای عملی:
--------------
• تابلوهای تبلیغاتی در فروشگاه‌ها، رستوران‌ها و مراکز تجاری
• نمایش اطلاعات در فرودگاه‌ها، ایستگاه‌های قطار و مترو
• تابلوهای راهنمایی در بیمارستان‌ها و مراکز درمانی
• نمایش منو و اطلاعات در رستوران‌ها و کافه‌ها
• تابلوهای اطلاع‌رسانی در ادارات و سازمان‌ها
• سیستم‌های نمایش اطلاعات در سالن‌های کنفرانس

معماری کلی:
-----------
سیستم از دو بخش اصلی تشکیل شده است:

1. Backend (سرور مرکزی):
   - Django REST API برای مدیریت داده‌ها
   - WebSocket برای ارتباط لحظه‌ای با نمایشگرها
   - سیستم احراز هویت و امنیت چندلایه
   - مدیریت کاربران با سطوح دسترسی مختلف

2. Frontend (پنل مدیریت):
   - رابط کاربری مدرن با Vue.js
   - به‌روزرسانی‌های لحظه‌ای از طریق WebSocket
   - مدیریت قالب‌ها، محتوا و برنامه‌های زمانی

3. Screen Client (نرم‌افزار روی نمایشگرها):
   - اتصال به سرور از طریق WebSocket یا HTTP
   - دریافت و اجرای دستورات
   - ارسال وضعیت و سلامت سیستم
   - دانلود و نمایش محتوا

جریان کاری کلی:
--------------
1. مدیر سیستم قالب‌های نمایشی را ایجاد می‌کند (Template)
2. محتوا (تصاویر، ویدیوها) به قالب‌ها اضافه می‌شود
3. نمایشگرها در سیستم ثبت می‌شوند و احراز هویت می‌شوند
4. قالب‌ها به نمایشگرها اختصاص داده می‌شوند
5. محتوا به صورت خودکار روی نمایشگرها دانلود می‌شود
6. نمایشگرها محتوا را نمایش می‌دهند
7. سیستم به صورت لحظه‌ای وضعیت نمایشگرها را نظارت می‌کند
8. مدیر می‌تواند دستورات را از راه دور ارسال کند (ریستارت، تغییر قالب)
9. برنامه‌های زمانی برای تغییر خودکار قالب‌ها تنظیم می‌شوند

================================================================================
A) HIGH-LEVEL CAPABILITIES
================================================================================

سیستم ScreenGram قابلیت‌های زیر را ارائه می‌دهد:

1. SCREEN MANAGEMENT
   - Register and manage physical display devices (Screens)
   - Real-time status monitoring (online/offline, heartbeat tracking)
   - Device authentication and security (auth_token, secret_key)
   - Health monitoring (CPU, memory, latency tracking)
   - IP address tracking and device information management

2. TEMPLATE & CONTENT MANAGEMENT
   - Create hierarchical templates (Template → Layer → Widget → Content)
   - Support for multiple content types (image, video, text, webview, chart, json)
   - Template activation on screens with automatic content synchronization
   - Content download and caching system with retry mechanisms
   - Template versioning and metadata management

3. COMMAND EXECUTION
   - Queue and execute commands on screens (restart, refresh, change_template, 
     display_message, sync_content, custom)
   - Priority-based command queuing
   - Command expiration and retry logic
   - Real-time command status tracking

4. SCHEDULING SYSTEM
   - Time-based template scheduling on screens
   - Support for one-time, daily, weekly, and monthly schedules
   - Priority-based conflict resolution
   - Automatic schedule execution via management commands

5. USER & ROLE MANAGEMENT
   - Custom user model with role-based access control (RBAC)
   - Five role levels: SuperAdmin, Admin, Operator, Manager, Viewer
   - Multi-tenant organization support
   - JWT-based authentication
   - Permission-based resource access

6. COMPREHENSIVE LOGGING
   - Automatic logging of screen status changes
   - Content download tracking with success/failure logs
   - Command execution logs with timing and error tracking
   - Analytics and reporting capabilities

7. API INTEGRATION
   - RESTful API for all operations
   - Screen-to-server communication endpoints (heartbeat, command pull, 
     command response, content sync)
   - Role-based API access control
   - Comprehensive serializers for data validation

8. WEBSOCKET COMMUNICATION
   - Real-time bidirectional communication with screens
   - WebSocket connection registry for active screen tracking
   - Automatic connection management and cleanup
   - Command delivery via WebSocket with HTTP fallback
   - Screen authentication during WebSocket handshake

9. CONTENT STORAGE SYSTEM
   - Support for Local File System and Amazon S3 storage backends
   - Automatic backend selection based on configuration
   - File type and size validation
   - Hash-based integrity checking (SHA-256)
   - Secure signed URLs for S3 content
   - File upload handling with metadata tracking

10. SECURITY FEATURES
    - HMAC-SHA256 signature generation and validation for commands
    - Timestamp validation (reject stale messages)
    - Nonce-based replay attack protection
    - Rate limiting per screen (configurable)
    - Secure payload signing for command delivery
    - Screen authentication via auth_token and secret_key

11. ADMIN DASHBOARD & REAL-TIME UPDATES
    - WebSocket-based admin dashboard for real-time monitoring
    - JWT authentication for dashboard connections
    - Role-based data filtering and organization isolation
    - Real-time command status updates
    - Live content sync progress tracking
    - Screen status and health monitoring
    - Automatic event broadcasting via Django signals

12. FRONTEND APPLICATION
    - Vue.js 3 frontend with modern UI
    - Real-time WebSocket integration
    - Pinia state management
    - Vue Router for navigation
    - Tailwind CSS for styling
    - Auto-reconnect WebSocket composable
    - Responsive design with Heroicons

13. LOG MANAGEMENT
    - Automated log cleanup management command
    - Log export functionality (CSV/JSON)
    - Date range filtering for exports
    - Screen-specific log filtering

H) TECHNICAL ARCHITECTURE
================================================================================

COMMUNICATION FLOW:
  1. Screen → WebSocket connection (ws://server/ws/screen/?auth_token=xxx&secret_key=yyy)
  2. ScreenConsumer authenticates و connection را register می‌کند
  3. Command queue می‌شود → Django signal broadcast می‌کند → Dashboard users به‌روزرسانی می‌گیرند
  4. Command.execute() → _send_to_screen() → WebSocket (اولویت) یا HTTP (fallback)
  5. Screen command را دریافت می‌کند، signature را validate می‌کند، اجرا می‌کند
  6. Screen status update می‌فرستد → Signal broadcast می‌کند → Dashboard به‌روزرسانی می‌گیرد

DASHBOARD REAL-TIME FLOW:
  1. User login → JWT token → Frontend WebSocket connection (ws://server/ws/dashboard/?token=...)
  2. AdminDashboardConsumer JWT را validate می‌کند → User به dashboard_updates group اضافه می‌شود
  3. User می‌تواند subscribe کند به screens/commands خاص
  4. Backend events خودکار broadcast می‌شوند → فیلتر بر اساس role و organization
  5. Frontend real-time updates دریافت می‌کند → Auto-reconnect با exponential backoff

CONTENT DELIVERY FLOW:
  1. Upload: API یا save_uploaded_file() → ContentStorageManager validation → Save (Local/S3)
  2. Hash calculation و storage برای integrity checking
  3. download_to_screen() → Secure URL generation (signed URL برای S3)
  4. Download command via Command system (WebSocket/HTTP) → Screen download می‌کند
  5. Hash verification → ContentDownloadLog creation → Status report via content-sync endpoint

SECURITY LAYERS:
  1. Authentication: auth_token + secret_key برای screen، JWT برای users
  2. Authorization: Role-based access control
  3. Message Integrity: HMAC-SHA256 signatures
  4. Replay Protection: Nonce-based tracking
  5. Timestamp Validation: 5-minute window
  6. Rate Limiting: 60 commands/minute per screen
  7. File Integrity: SHA-256 hash verification
  8. Secure URLs: Signed URLs با expiration

STORAGE BACKENDS:
  • Local File System: MEDIA_ROOT/content/{content_id}/ - مناسب برای development
  • Amazon S3: Private bucket با signed URLs - مناسب برای production و multi-region
